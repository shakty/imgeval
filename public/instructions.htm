Task: Employment Identification (test)

The following figure shows an overview of an assisted facial recognition system that aims to identify if a person entering the building is an employee or not.

When a person enters the building, there is a physical barrier in front of which there is a security camera. On the other side, there is a security guard with a screen monitor in front. In this study, you are playing the role of the security guard.

The security camera captures an image of the entering person. We will refer to this image as the probe. The Facial Recognition System (FRS) has access to a gallery containing photos of the employees. The FRS compares the probe image with the photos in the gallery. If FRS finds a match, the barrier is automaticaly removed. Otherwise, the FRS shows the photo of the most similiar profile photo in the gallery aside with the probe image on the security guard's screen. The security guard compares the two images, and if he/she decides that they belong to the same person he/she grants access. Otherwise, he/she asks the entering person to provide second factor iauthentication such as ID card.

We refer to Facial Recognition System (FRS) as a technology which is capable of identifying the similarity score between two images. Similarity score is a number between 0 and 1 indicating how similar are two images. Similarity score of 0 indicates that two images are very different and similarity score of 1 indicates that two images are very similar. One of the parameters of an FRS, is the threshold value. The pair of images which have similarity score higher than the threshold value are identified as a match.

You are playing the role of the security guard. We will show you pairs of images along with their similarity score and you should identify if two images have the same identity i.e., the entering person is an employee, or not. Note that a wrong decision could result in granting access to a non-employee and therefore having an unauthorized person in the building. Remember that the image pairs that you see are the ones for which the similarity score is below the FRS's threshold value and therefore the system could not identify a match automatically. Here is an example

  • Similarity score : .92
  • FRS threshold value to accept as a match: .98
